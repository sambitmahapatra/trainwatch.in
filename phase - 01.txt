# Phase-01 Architecture

**Project: TrainWatch (Working Name)**
Purpose: Notify developers when ML training **completes or fails**, with a **minimal run summary**.

---

# 1?? System Overview

Phase-01 architecture is intentionally simple.

```
User Training Script
        Î
        ?
TrainWatch Monitor
        Î
        +-- Runtime Tracker
        +-- Metric Logger (optional)
        +-- Error Capture
        Î
        ?
Summary Generator
        Î
        ?
Notification Engine
        Î
        +-- Email
        +-- Telegram
```

The system only answers one question:

> **Did training finish or fail?**

And sends a **short summary**.

---

# 2?? Core Design Principles

### Simplicity

Developers should integrate the tool in **two lines of code**.

### Framework-Agnostic

Works with:

* PyTorch
* TensorFlow
* scikit-learn
* custom training loops

### Minimal Dependencies

Avoid heavy libraries.

### Reliability

Notifications must always be sent if training finishes or crashes.

# 4?? Core Modules

## 4.1 Monitor Module

This is the **entry point**.

Responsible for:

* starting monitoring
* stopping monitoring
* capturing exceptions

Example usage:

```python
from trainwatch import monitor

monitor.start()

train_model()

monitor.end()
```

Internal behavior:

```
start()
   ?
record start_time
   ?
execute training
   ?
capture status
   ?
generate summary
   ?
send notification
```

---

# 4.2 Runtime Tracker

Tracks execution time.

Example output:

```
Training runtime: 2h 13m
```

Implementation logic:

```
start_time = time.now()
end_time = time.now()

runtime = end_time - start_time
```

---

# 4.3 Metric Logger (Optional)

Allows users to log metrics manually.

Example:

```python
monitor.log(epoch=10, loss=0.23, val_acc=0.91)
```

Internally stored as:

```
metrics = [
 {epoch:1, loss:0.72},
 {epoch:2, loss:0.61}
]
```

Used later for summary.

---

# 4.4 Error Capture System

Critical for detecting training failures.

Implementation:

```
try:
   train_model()
except Exception as e:
   monitor.fail(e)
```

Captured data:

```
error type
error message
last epoch
runtime
```

---

# 4.5 Summary Generator

Produces **4-5 line report**.

Example output:

```
Training Completed

Runtime: 2h 13m
Best Validation Accuracy: 0.93
Epoch of Best Model: 16
Final Loss: 0.28
```

Or failure case:

```
Training Failed

Runtime: 32m
Error: CUDA Out Of Memory
Last Epoch: 5
```

This module formats messages for notifications.

---

# 4.6 Notification Engine

Responsible for sending alerts.

Interface design:

```
send(message)
```

Supports multiple channels.

```
EmailNotifier
TelegramNotifier
```
